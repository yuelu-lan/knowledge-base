# Claude Code 记忆管理：CLAUDE.md、@import 与 .claude/rules/

> 每个 Claude Code 会话都从全新上下文开始。要让 Claude 跨会话记住项目约定，靠三种机制：`CLAUDE.md` 文件、`@path/to/import` 导入、`.claude/rules/` 目录规则。
>
> 参考：
>
> - 英文：<https://code.claude.com/docs/en/memory>
> - 中文：<https://code.claude.com/docs/zh-CN/memory>

---

## 一、CLAUDE.md 文件

为 Claude 提供持久指令的纯文本 markdown，每次会话开始时加载。

### 1.1 放在哪里（按加载顺序，越靠后优先级越高）

| 范围 | 位置 | 共享对象 |
| :-- | :-- | :-- |
| 托管策略 | macOS: `/Library/Application Support/ClaudeCode/CLAUDE.md`<br>Linux/WSL: `/etc/claude-code/CLAUDE.md` | 组织所有用户 |
| 用户指令 | `~/.claude/CLAUDE.md` | 仅你（所有项目） |
| 项目指令 | `./CLAUDE.md` 或 `./.claude/CLAUDE.md` | 团队（版本控制） |
| 本地指令 | `./CLAUDE.local.md`（加入 `.gitignore`） | 仅你（当前项目） |

### 1.2 加载顺序

从**当前工作目录向上遍历**目录树，沿途拼接所有发现的 `CLAUDE.md` 和 `CLAUDE.local.md`，**不互相覆盖而是叠加**。越接近启动位置的指令越晚被读取。

- 每个目录中，`CLAUDE.local.md` 在 `CLAUDE.md` 之后附加（个人笔记最后读）。
- 子目录中的文件**不在启动时加载**，而是 Claude 读取该子目录文件时按需加载。

### 1.3 编写原则

- **大小**：每个文件 < 200 行，太长消耗上下文、降低遵守度。
- **结构**：用标题和项目符号分组，别写密集长文。
- **具体性**：写"使用 2 空格缩进"，别写"正确格式化代码"。
- **一致性**：定期审查，删除冲突或过时指令。

> `/init` 可自动生成起始 CLAUDE.md。

## 二、`@path/to/import` 导入语法

CLAUDE.md 可用 `@path/to/import` 导入其他文件，**启动时展开**并随引用它的 CLAUDE.md 一起进入上下文。

### 2.1 基本用法

```text
有关项目概述，请参阅 @README，npm 命令请参阅 @package.json。

# 其他指令
- git 工作流 @docs/git-instructions.md
```

### 2.2 规则

- 相对路径相对于「包含导入的文件」解析，非工作目录。
- 递归导入最大深度 **4 跳**。
- 跨 worktree 共享个人指令：从主目录导入 `@~/.claude/my-project-instructions.md`。
- **反引号防误导入**：用反引号包住 @README（写成 `@README`）时，`@` 不触发导入，只保留字面文本；直接写 @README（不加反引号）才会被导入。

### 2.3 注意事项

- **首次外部导入需批准**：Claude Code 会弹出对话框列出文件，拒绝则导入禁用。
- **与 AGENTS.md 共存**：Claude 只读 CLAUDE.md，可用 `@AGENTS.md` 导入复用，或 `ln -s AGENTS.md CLAUDE.md`。
- **导入 ≠ 省上下文**：拆导入利于组织，但导入文件仍启动加载。要省上下文用路径范围规则或 skill。
- **导入是"启动时确定性注入"**：`@` 让文件在会话启动时一定进入上下文（确定性），但进入上下文后仍属"上下文而非强制配置"——Claude 会尽量遵循但不保证严格遵守。注意区分「确定性加载」与「强制遵循」两层。

## 三、`.claude/rules/` 目录规则

把指令拆成多个主题文件，且可**按文件路径范围限定**——仅在 Claude 处理匹配文件时才加载。

### 3.1 目录结构

```text
.claude/
├── CLAUDE.md
└── rules/
    ├── code-style.md
    ├── testing.md
    └── security.md
```

所有 `.md` 文件递归发现，可用子目录组织。**无 `paths` frontmatter 的规则**在启动时加载，优先级与 `.claude/CLAUDE.md` 相同。

### 3.2 路径范围规则（核心能力）

用 `paths` 字段限定，仅在处理匹配文件时触发：

```markdown
---
paths:
  - 'src/api/**/*.ts'
---

# API 开发规则

- 所有端点必须包括输入验证
```

glob 模式速查：

| 模式                | 匹配                       |
| :------------------ | :------------------------- |
| `**/*.ts`           | 任意目录的 TypeScript 文件 |
| `src/**/*`          | `src/` 下所有文件          |
| `src/**/*.{ts,tsx}` | 多扩展名（大括号扩展）     |

### 3.3 共享与用户级

- **跨项目共享**：`.claude/rules/` 支持符号链接，可链接共享目录或单个文件。
- **用户级规则**：`~/.claude/rules/` 适用于机器上每个项目，在项目规则之前加载（项目规则优先级更高）。

### 3.4 rules vs skills

- **rules**：每次会话（无 `paths`）或打开匹配文件（有 `paths`）时加载进上下文。
- **skills**：仅在你调用或 Claude 判定相关时才加载，最省上下文。

不需要始终在上下文中的特定任务指令，用 skill。

## 四、补充技巧：仅写路径，由 Claude 按需读取

> ⚠️ **非官方机制**：官方文档只记载了 `@import`（启动时导入）和反引号抑制导入两种行为，并未把"纯路径文本"列为记忆机制。本节是实践做法，依赖 Claude 读到文本后的自然行为，**无解析器保证**。

与 `@import`（启动时确定性注入上下文）相对的另一种做法：**在 CLAUDE.md 里只写文件路径作为文本提示，不加 `@`**。路径只是作为"这里有相关资料"的信息被 Claude 看到，Claude 在认为需要时用 Read 工具自行读取，不需要时不加载——零启动开销。

```text
# 参考资料（按需查阅，非必读）
- 项目架构说明：docs/architecture.md
- 历史决策记录：docs/adr/
- 复杂业务规则：docs/business-rules.md
```

### 4.1 实测验证

在干净会话中做了对照实验（CLAUDE.md 用纯路径指向 docs 下的文件）：

| 场景 | Claude 读取行为 | 结果 |
| :-- | :-- | :--- |
| 问架构/业务规则（任务相关） | 主动 Read 了纯路径指向的文件 | ✅ 正确答出内容 |
| 问项目名（CLAUDE.md 已有答案） | 仅读 CLAUDE.md，未碰 docs | ✅ 不多读 |

结论：纯路径文本确实会被当作资料提示，**按需**触发读取。

### 4.2 `@import` vs 纯路径文本

| 写法 | 加载行为 | 确定性 | 上下文开销 | 适用场景 |
| :-- | :-- | :-- | :-- | :-- |
| `@docs/architecture.md` | 启动时确定性注入上下文 | 高（一定加载） | 启动即消耗 | 每次会话都相关的核心资料 |
| `docs/architecture.md`（纯文本） | Claude 判断需要时自行 Read | 弱（按需，无保证） | 零启动开销 | 体量大、偶尔才用的参考文档 |

### 4.3 适用判断与局限

- 文档**每次都相关**且体积小 → 用 `@import`。
- 文档**只在特定任务才用得上**或体量较大（完整架构说明、ADR 列表）→ 仅写路径，让 Claude 按需读。
- 想在 CLAUDE.md 里**提及一个 `@` 路径但不导入它** → 用反引号包住，写成 `@README`，`@` 不触发导入。（反引号只对 `@` 导入有意义；不带 `@` 的纯路径本来就不会被自动导入。）

**局限**：

- 依赖 Claude 判断"是否需要"，换问法/换会话不保证一定去读。
- 依赖 Read 工具权限，环境受限时可能读不到。
- 官方未记载，未来版本行为可能变化。

## 三者对比

> 下表只列三个官方记忆机制。纯路径文本是实践技巧，见第四节。

| 机制 | 加载时机 | 上下文开销 | 典型场景 |
| :-- | :-- | :-- | :-- |
| **CLAUDE.md** | 每次会话启动（向上遍历 + 子目录按需） | 完整加载（建议 <200 行） | 全局/项目级标准、工作流 |
| **`@path` 导入** | 启动时确定性注入上下文 | 启动即消耗 | 每次都相关的核心资料 |
| **rules（无 paths）** | 启动时，与 `.claude/CLAUDE.md` 同级 | 启动加载 | 主题化拆分大 CLAUDE.md |
| **rules（有 paths）** | 处理匹配文件时才加载 | 按需，省上下文 | 仅特定目录/文件类型生效的规则 |

---

## 关键提醒

- CLAUDE.md 和规则都是**上下文而非强制配置**；硬性要求用 [hook](https://code.claude.com/docs/zh-CN/hooks-guide)。
- 排查指令没生效，先用 `/memory` 确认文件是否被加载。
- 要省上下文：用 `paths` 范围规则或 skill，而不是 `@import`。
