# MySQL

## 基本概念

- **数据库（DataBase，DB）**：存储和管理数据的仓库
- **数据库管理系统（DBMS）**：操作和管理数据库的软件，如 MySQL、Oracle、PostgreSQL
- **SQL（Structured Query Language）**：操作关系型数据库的统一标准语言

三者关系：程序员 → 发送 SQL 语句 → DBMS → 操作 → 数据库中的数据。所有关系型数据库都通过 SQL 操作，学会 SQL 就能操作任何关系型数据库。

| 数据库 | 特点 |
| --- | --- |
| Oracle | 大型收费数据库，功能强大，价格昂贵 |
| MySQL | 开源免费，互联网公司首选 |
| SQL Server | 微软出品，C#/.NET 常用 |
| PostgreSQL | 开源免费，功能丰富 |
| DB2 | IBM 大型数据库 |
| SQLite | 嵌入式微型数据库，Android 内置 |
| MariaDB | MySQL 的开源分支，兼容性好 |

关系型数据库（RDBMS）基于**二维表**存储数据，格式统一、SQL 标准统一、支持复杂查询。数据模型：一个数据库服务器 → 多个数据库 → 多张表 → 多行记录。

连接 MySQL：`mysql -u用户名 -p密码 [-h服务器IP -P端口号]`，`-h` 不填默认连接本地 127.0.0.1，`-P` 不填默认端口 3306。

## SQL 分类

SQL 语句按功能分为四大类：

| 分类 | 全称 | 说明 |
| --- | --- | --- |
| **DDL** | Data Definition Language | 数据定义语言，定义数据库对象（数据库、表、字段） |
| **DML** | Data Manipulation Language | 数据操作语言，对表中数据进行增删改 |
| **DQL** | Data Query Language | 数据查询语言，查询表中的记录 |
| **DCL** | Data Control Language | 数据控制语言，创建数据库用户、控制访问权限 |

## DDL —— 数据定义语言

**数据库操作**：

```sql
show databases;                                                    -- 查询所有数据库
select database();                                                 -- 查询当前数据库
create database [if not exists] 数据库名 [default charset utf8mb4]; -- 创建数据库（MySQL8 默认字符集已是 utf8mb4，可不指定）
use 数据库名;                                                       -- 使用数据库
drop database [if exists] 数据库名;                                 -- 删除数据库
```

同名数据库不能重复创建，删除不存在的数据库会报错，均可用 `if not exists` / `if exists` 规避。

**创建表**：

```sql
create table 表名(
    字段1  字段1类型 [约束]  [comment 字段1注释],
    字段2  字段2类型 [约束]  [comment 字段2注释],
    ......
    字段n  字段n类型 [约束]  [comment 字段n注释]
) [comment 表注释];
```

```sql
create table tb_user (
    id int comment 'ID,唯一标识',
    username varchar(20) comment '用户名',
    name varchar(10) comment '姓名',
    age int comment '年龄',
    gender char(1) comment '性别'
) comment '用户表';
```

上例中 id 字段虽然备注为"唯一标识"，但数据库层面并未限制其数据，仍可插入重复值——要真正限制字段数据，需要用到**约束**。

**约束**：MySQL 提供 5 种约束，作用于表字段上，用于保证数据的正确性、有效性和完整性，可在建表或改表时添加。

| 约束 | 描述 | 关键字 |
| --- | --- | --- |
| 非空约束 | 字段值不能为 null | `not null` |
| 唯一约束 | 字段值唯一不重复 | `unique` |
| 主键约束 | 唯一标识，非空且唯一 | `primary key` |
| 默认约束 | 未指定值时采用默认值 | `default` |
| 外键约束 | 建立表间数据连接 | `foreign key` |

`auto_increment`（主键自增）：每次插入新行时数据库自动生成主键值，从 1 开始递增，避免手动维护唯一 id 的麻烦。

```sql
create table tb_user (
    id int primary key auto_increment comment 'ID,唯一标识',  -- 主键且自增
    username varchar(20) not null unique comment '用户名',   -- 非空且唯一
    name varchar(10) not null comment '姓名',                -- 非空
    age int comment '年龄',
    gender char(1) default '男' comment '性别'               -- 默认值
) comment '用户表';
```

**数据类型**：MySQL 数据类型分三类：数值、字符串、日期时间。

数值类型：

| 类型 | 大小 | 有符号范围 | 无符号范围 | 描述 |
| --- | --- | --- | --- | --- |
| TINYINT | 1byte | (-128，127) | (0，255) | 小整数值 |
| SMALLINT | 2bytes | (-32768，32767) | (0，65535) | 大整数值 |
| MEDIUMINT | 3bytes | (-8388608，8388607) | (0，16777215) | 大整数值 |
| INT/INTEGER | 4bytes | (-2147483648，2147483647) | (0，4294967295) | 大整数值 |
| BIGINT | 8bytes | (-2^63，2^63-1) | (0，2^64-1) | 极大整数值 |
| FLOAT | 4bytes | 约 ±3.4E+38 | 0 和约 3.4E+38 | 单精度浮点数 |
| DOUBLE | 8bytes | 约 ±1.8E+308 | 0 和约 1.8E+308 | 双精度浮点数 |
| DECIMAL | 依赖 M(精度)/D(标度) | — | — | 精确定点数（小数值） |

```sql
age tinyint unsigned,     -- 年龄：不会是负数，数值不大
score double(4,1)         -- 分数：总分100，最多一位小数
```

字符串类型：

| 类型 | 大小 | 描述 |
| --- | --- | --- |
| CHAR | 0-255 bytes | 定长字符串（需指定长度，性能更高） |
| VARCHAR | 0-65535 bytes | 变长字符串（指定长度为最大占用长度） |
| TINYBLOB / TINYTEXT | 0-255 bytes | 短二进制/文本数据 |
| BLOB / TEXT | 0-65535 bytes | 二进制/长文本数据 |
| MEDIUMBLOB / MEDIUMTEXT | 0-16777215 bytes | 中等长度二进制/文本数据 |
| LONGBLOB / LONGTEXT | 0-4294967295 bytes | 极大二进制/文本数据 |

```sql
username varchar(50),  -- 用户名：长度不定，最长不超过50
phone char(11)         -- 手机号：固定长度11
```

日期时间类型：

| 类型 | 大小 | 范围 | 格式 | 描述 |
| --- | --- | --- | --- | --- |
| DATE | 3 | 1000-01-01 至 9999-12-31 | YYYY-MM-DD | 日期值 |
| TIME | 3 | -838:59:59 至 838:59:59 | HH:MM:SS | 时间值 |
| YEAR | 1 | 1901 至 2155 | YYYY | 年份值 |
| DATETIME | 8 | 1000-01-01 至 9999-12-31 | YYYY-MM-DD HH:MM:SS | 日期+时间 |
| TIMESTAMP | 4 | 1970-01-01 至 2038-01-19 | YYYY-MM-DD HH:MM:SS | 日期+时间（时间戳） |

```sql
birthday date,          -- 生日：只需年月日
create_time datetime    -- 创建时间：需精确到时分秒
```

**表结构其他操作**：

```sql
-- 准备demo表
create table emp(
    id int unsigned primary key auto_increment comment 'ID,主键',
    username varchar(20) not null unique comment '用户名',
    name varchar(10) not null comment '姓名',
    gender tinyint unsigned not null comment '性别, 1:男, 2:女',
    phone char(11) not null unique comment '手机号',
    entry_date date comment '入职日期'
) comment '员工表';

show tables;                 -- 查询当前数据库的所有表
desc emp;                    -- 查看表结构：字段、类型、是否可空、默认值等
show create table emp;       -- 查询建表语句

alter table emp add qq varchar(11) comment 'QQ号码';               -- 添加字段
alter table emp modify qq varchar(13) comment 'QQ号码';            -- 修改字段类型
alter table emp change qq qq_num varchar(13) comment 'QQ号码';     -- 改字段名+类型
alter table emp drop qq_num;                                       -- 删除字段
rename table emp to tb_emp;                                        -- 修改表名
drop table if exists tb_emp;                                       -- 删除表（数据一并删除）
```

工作中表结构的查看/修改/删除一般直接用图形化界面操作。

## DML —— 数据操作语言

对表中数据进行增（INSERT）、删（DELETE）、改（UPDATE）。

**增加**：

```sql
insert into 表名 (字段名1, 字段名2) values (值1, 值2);              -- 指定字段
insert into 表名 values (值1, 值2, ...);                           -- 全部字段
insert into 表名 (字段名1, 字段名2) values (值1, 值2), (值1, 值2);   -- 指定字段批量插入
insert into 表名 values (值1, 值2, ...), (值1, 值2, ...);           -- 全部字段批量插入
```

```sql
insert into emp(username, name, gender, phone, entry_date)
values ('wuji', '张无忌', 1, '13309091231', '2020-01-01');
```

注意事项：字段顺序需与值顺序一一对应，字符串和日期需加引号，且插入值需在字段类型规定范围内。

**修改**：

```sql
update 表名 set 字段名1 = 值1, 字段名2 = 值2, ... [where 条件];
```

```sql
update emp set name = '张三', entry_date = '2020-02-01' where id = 1;
```

没有 where 条件会修改整表所有数据；修改时一般同步更新 `update_time` 等公共字段为当前时间。

**删除**：

```sql
delete from 表名 [where 条件];
```

```sql
delete from emp where id = 1;
```

没有 where 条件会删除整表所有数据；DELETE 不能只删除某个字段的值（需用 UPDATE 将该字段置为 NULL）。

## DQL —— 数据查询语言

查询关键字 **SELECT**，完整语法结构：

```sql
SELECT 字段列表
FROM 表名列表
WHERE 条件列表
GROUP BY 分组字段列表
HAVING 分组后条件列表
ORDER BY 排序字段列表
LIMIT 分页参数
```

测试数据（精简 emp 表）：

```sql
create table emp(
    id int unsigned primary key auto_increment comment 'ID,主键',
    name varchar(10) not null comment '姓名',
    gender tinyint unsigned not null comment '性别, 1:男, 2:女',
    job tinyint unsigned comment '职位, 1:班主任, 2:讲师, 3:学工主管',
    salary int unsigned comment '薪资',
    entry_date date comment '入职日期'
) comment '员工表';

insert into emp(id, name, gender, job, salary, entry_date) values
(1, '张翠山', 1, 2, 9200, '2005-08-01'),
(2, '张三丰', 1, 1, 5300, '2010-01-01'),
(3, '俞莲舟', 1, 2, 8900, '2015-03-21'),
(4, '赵敏',   2, 3, 9500, '2008-01-01'),
(5, '张无忌', 1, NULL, NULL, '2018-01-01');  -- 未分配职位
```

**基本查询**：

```sql
select 字段1, 字段2 from 表名;             -- 查询多个字段
select * from 表名;                        -- 查询所有字段（尽量少用，不直观、影响效率）
select 字段1 as 别名1, 字段2 as 别名2 from 表名; -- 设置别名
select distinct 字段列表 from 表名;         -- 去除重复记录
```

```sql
select name, entry_date from emp;
select distinct job from emp;  -- 查询已有员工关联了哪几种职位
```

**条件查询**：语法为 `select 字段列表 from 表名 where 条件列表`，条件由比较运算符和逻辑运算符构建。

比较运算符：

| 运算符 | 功能 |
| --- | --- |
| `>` `>=` `<` `<=` `=` | 大于/大于等于/小于/小于等于/等于 |
| `<>` 或 `!=` | 不等于 |
| `between ... and ...` | 在某范围之内（含最小、最大值） |
| `in(...)` | 在列表中的值，多选一 |
| `like 占位符` | 模糊匹配（`_` 匹配单个字符，`%` 匹配任意个字符） |
| `is null` | 是 null（不能用 `= null`） |

逻辑运算符：

| 运算符 | 功能 |
| --- | --- |
| `and` 或 `&&` | 并且（多个条件同时成立） |
| `or` 或 `\|\|` | 或者（多个条件任意一个成立） |
| `not` 或 `!` | 非，不是 |

```sql
-- between：查询入职日期在 2005-01-01 到 2010-01-01 之间的员工
select name, entry_date from emp where entry_date between '2005-01-01' and '2010-01-01';

-- in：查询职位是讲师(2)或班主任(1)的员工
select name, job from emp where job in (1, 2);

-- like：查询姓名为两个字（'__'）或姓张（'张%'）的员工
select name from emp where name like '__';
select name from emp where name like '张%';
```

**聚合函数**：对一列数据做纵向计算并返回单个结果，会自动忽略 NULL 值。

| 函数 | 功能 |
| --- | --- |
| `count` | 统计数量 |
| `max` | 最大值 |
| `min` | 最小值 |
| `avg` | 平均值 |
| `sum` | 求和 |

```sql
select count(*) from emp;         -- 统计员工数量，count(*) 是推荐写法（MySQL 底层有优化）
select count(job) from emp;       -- 4（job 列有一个 NULL，不计入）
select avg(salary), max(salary), min(salary), sum(salary) from emp;
```

**分组查询**：按某列或几列把相同数据合并，通常配合聚合函数计算。

```sql
select 字段列表 from 表名 [where 条件] group by 分组字段名 [having 分组后过滤条件];
```

```sql
select gender, count(*) from emp group by gender;  -- 按性别分组统计人数

select job, count(*) from emp
where job is not null
group by job
having count(*) >= 2;  -- 分组后过滤：只看人数≥2的职位
```

分组之后一般只查询聚合函数和分组字段；执行顺序为 where → 聚合函数 → having。where 在分组前过滤且不能判断聚合函数，having 在分组后过滤且可以判断聚合函数。

**排序查询**：

```sql
select 字段列表 from 表名 [where 条件] [group by 分组字段] order by 字段1 排序方式1, 字段2 排序方式2 ...;
```

排序方式 `ASC`（升序，默认可省略）、`DESC`（降序）。

```sql
select name, entry_date from emp order by entry_date desc;              -- 按入职时间降序
select name, entry_date from emp order by entry_date asc, id desc;      -- 多字段排序：第一字段相同时按第二字段排
```

**分页查询**：

```sql
select 字段列表 from 表名 limit 起始索引, 查询记录数;
```

```sql
select name from emp limit 0, 2;  -- 第1页，每页2条：从索引0开始取2条
select name from emp limit 2, 2;  -- 第2页，每页2条：从索引2开始取2条
select name from emp limit 2;     -- 第1页可省略起始索引，简写为 limit 条数
```

起始索引从 0 开始，计算公式为 `起始索引 = (页码 - 1) * 每页记录数`；分页是数据库方言，MySQL 中用 `LIMIT`。

## 多表查询

```sql
-- 数据源准备
create table classes (
    id      int primary key auto_increment,
    name    varchar(64) not null comment '班级名称'
);

create table students (
    id       int primary key auto_increment,
    name     varchar(64) not null comment '学生姓名',
    class_id int comment '班级ID（NULL表示未分班）'
);

insert into classes (name) values ('一班'), ('二班'), ('三班');

insert into students (name, class_id) values
('张三', 1),
('李四', 1),
('王五', 2),
('周八', 2),
('赵六', NULL),  -- 未分班
('孙七', NULL);  -- 未分班
```

`INNER JOIN`（内连接）：只返回两表都能匹配的行（交集）。

```sql
select s.name, c.name as class_name
from students s
inner join classes c on s.class_id = c.id;
```

`LEFT JOIN`（左连接）：返回左表全部行，右表无匹配则为 NULL。

```sql
select s.name, c.name as class_name
from students s
left join classes c on s.class_id = c.id;
```

`RIGHT JOIN`（右连接）：返回右表全部行，左表无匹配则为 NULL。

```sql
select s.name, c.name as class_name
from students s
right join classes c on s.class_id = c.id;
```

`CROSS JOIN`（笛卡尔积）：左表每行 × 右表每行，结果行数 = 两表行数之积。

```sql
select s.name, c.name as class_name
from students s
cross join classes c;
```

| 类型 | 返回行数 | 未匹配行 | 适用场景 |
| --- | --- | --- | --- |
| **INNER JOIN** | 4行 | 不显示 | 只要有关联关系的数据 |
| **LEFT JOIN** | 6行 | 右表（赵六、孙七的班级）显示 NULL | 查左表全部，顺带关联右表（最常用） |
| **RIGHT JOIN** | 5行 | 左表（三班没有学生）显示 NULL | 查右表全部，顺带关联左表（少用） |
| **CROSS JOIN** | 18行 | 全部组合 | 生成组合数据，日常业务慎用 |
