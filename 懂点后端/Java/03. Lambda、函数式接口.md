## Lambda 表达式

- Lambda 本质上是「实现了函数式接口的匿名函数」
- 编译器会自动推断你在实现哪个接口的哪个方法

```java
// Java 7 写法：匿名内部类
fruits.sort(new Comparator<String>() {
    @Override
    public int compare(String a, String b) { return a.compareTo(b); }
});

// Java 8+ 写法：Lambda（等价于 JS 箭头函数）
// JS: fruits.sort((a, b) => b.localeCompare(a));
fruits.sort((a, b) -> b.compareTo(a));
```

## 函数式接口

### `Function<T, R>` —— 转换函数

`Function` 是最基础的函数式接口：**输入 T，输出 R**。

| 接口            | 方法                | 描述                              |
| --------------- | ------------------- | --------------------------------- |
| `Function`      | `R apply(T t)`      | 输入一个值，返回另一个值          |
| `BiFunction`    | `R apply(T t, U u)` | 输入两个值，返回一个值            |
| `UnaryOperator` | `T apply(T t)`      | 输入输出同类型（Function 的特例） |

```java
Function<String, Integer> strLen = String::length;
strLen.apply("hello"); // 5

Function<String, String> toUpper = String::toUpperCase;
toUpper.apply("java"); // "JAVA"

BiFunction<String, Integer, String> repeat = (s, n) -> s.repeat(n);
repeat.apply("ab", 3); // "ababab"

UnaryOperator<String> trim = String::trim;
trim.apply("  hello  "); // "hello"

// 在 Stream 中使用
List<Integer> lengths = words.stream().map(strLen).collect(Collectors.toList());
```

### `Predicate<T>` —— 条件判断

`Predicate` 专门用于条件判断：输入 T，返回 boolean。

强大的逻辑组合方法：

| 方法         | 等价       | JS                         |
| ------------ | ---------- | -------------------------- | --- | ----------------- |
| `p1.and(p2)` | `p1 && p2` | 手写 `s => p1(s) && p2(s)` |
| `p1.or(p2)`  | `p1        |                            | p2` | 手写 `s => p1(s)` |
| `p.negate()` | `!p`       | 手写 `s => !p(s)`          |

- 最常用于 Stream 的 `.filter()` 操作

```java
Predicate<String> isLong = s -> s.length() > 5;
Predicate<String> startsWithJ = s -> s.startsWith("j");

Predicate<String> longAndJ = isLong.and(startsWithJ);
Predicate<String> longOrJ  = isLong.or(startsWithJ);
Predicate<String> notLong  = isLong.negate();

List<String> words = List.of("java", "javascript", "go", "python", "js");
words.stream().filter(longAndJ).collect(Collectors.toList()); // [javascript]
words.stream().filter(longOrJ).collect(Collectors.toList());  // [java, javascript, js]
words.stream().filter(notLong).collect(Collectors.toList());  // [java, go, js]
```

### `Consumer<T>` —— 消费者（无返回值）

`Consumer` 的特点是**只消费，不产出**：输入 T，无返回值（void）。

常见用途：打印日志、发送通知、修改外部状态（副作用操作）。

`andThen`：`Consumer` 支持 `andThen` 链式组合，依次执行多个消费动作。

- `forEach(consumer)` 是最典型的使用场景

```java
Consumer<String> print = System.out::println;
print.accept("hello"); // 打印 hello

Consumer<String> prettyPrint = s -> System.out.println("[INFO] " + s);

// andThen 链式组合，依次执行
Consumer<String> both = print.andThen(prettyPrint);
both.accept("消息"); // 先打印 "消息"，再打印 "[INFO] 消息"

// forEach 是最典型的使用场景
products.forEach(p -> System.out.printf("%s: ¥%.1f%n", p.name(), p.price()));
```

### `Supplier<T>` —— 供应商（无参数工厂）

`Supplier` 是 Function 的反面：**无输入，有输出**，常用于工厂模式和惰性求值。

```java
Supplier<String> getGreeting = () -> "Hello, World!";
getGreeting.get(); // "Hello, World!"

// 惰性求值：定义时不执行，调用 get() 时才创建
Supplier<List<String>> lazyList = () -> Arrays.asList("a", "b", "c");

// 常用于 getOrDefault 模式
static <T> T getOrDefault(T value, Supplier<T> defaultSupplier) {
    return value != null ? value : defaultSupplier.get();
}
getOrDefault(null, () -> "默认值");   // "默认值"
getOrDefault("已有值", () -> "忽略"); // "已有值"
```

### 自定义 `@FunctionalInterface`

任何只有 **1 个抽象方法**的接口都是函数式接口，可以用 Lambda 实现。

`@FunctionalInterface` 注解是可选的，但强烈推荐加上。

Java 标准库中的 `Function`、`Predicate`、`Consumer`、`Supplier` 都是 `@FunctionalInterface`。

```java
@FunctionalInterface
interface StringTransformer {
    String transform(String input); // 只有 1 个抽象方法
}

// 用 Lambda 实现，传入不同行为
static void applyAll(List<String> words, StringTransformer fn) {
    words.forEach(w -> System.out.println(w + " → " + fn.transform(w)));
}

applyAll(words, s -> s.toUpperCase()); // 传 Lambda
applyAll(words, String::trim);         // 传方法引用
```

### 函数组合（andThen / compose）

`Function` 接口提供两个方法用于**函数组合**，对应函数式编程中的 pipe 和 compose：

| 方法           | 执行顺序   | 等价      | JS 对比         |
| -------------- | ---------- | --------- | --------------- |
| `f.andThen(g)` | 先 f，再 g | `g(f(x))` | `pipe(f, g)`    |
| `f.compose(g)` | 先 g，再 f | `f(g(x))` | `compose(f, g)` |
