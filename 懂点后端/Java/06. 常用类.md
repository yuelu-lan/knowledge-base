# 常用类

## Object

Object 是 Java 类层次最顶层的基类，所有类都直接或间接继承自 Object，属于 `java.lang` 包（无需导包），构造方法为 `public Object()`。

| 方法 | 说明 |
| --- | --- |
| `hashCode()` | 返回对象哈希码值（通过地址值计算），同一对象返回值相同，主要用于提升哈希表性能 |
| `getClass()` | 返回字节码文件对象（Class 对象），格式为 `class 包名.类名` |
| `toString()` | 默认返回 `类的全限定名@哈希码十六进制`，直接输出对象时默认调用，通常需要重写 |
| `equals()` | 默认比较两个对象的**引用**（地址）；`==` 可比较基本类型（值）和引用类型（地址），equals 只能比较对象，默认同 `==`，通常重写为值比较 |

实际开发中通常需要重写 `equals()` 和 `toString()`：

```Java
class Animal {
    private String speciesName;
    private int averageAge;

    @Override
    public boolean equals(Object o) {
        if (this == o) return true;
        if (o == null || getClass() != o.getClass()) return false;
        Animal animal = (Animal) o;
        return averageAge == animal.averageAge &&
                speciesName.equals(animal.speciesName);
    }

    @Override
    public String toString() {
        return "Animal{" +
                "speciesName='" + speciesName + '\'' +
                ", averageAge=" + averageAge +
                '}';
    }
}
```

## Objects

`Objects` 是 Java 7 引入的工具类，提供对象操作的实用静态方法，主要解决 null 安全问题。

|          | `Object`                 | `Objects`                    |
| -------- | ------------------------ | ----------------------------- |
| 包路径   | `java.lang.Object`       | `java.util.Objects`          |
| 引入版本 | Java 1.0                 | Java 7                       |
| 类型     | 普通类（所有类的根父类） | 工具类（final，不可继承）    |
| 用途     | 提供对象的基础方法       | 提供 null 安全的静态工具方法 |
| 实例化   | 可以实例化               | 不能实例化（私有构造器）     |

```Java
import java.util.Objects;

Objects.equals(a, b);           // null 安全的 equals，避免 NPE
Objects.toString(obj);          // null 时返回 "null"
Objects.toString(obj, "默认值"); // null 时返回指定默认值
Objects.hashCode(obj);          // null 时返回 0
Objects.hash(name, age, email); // 生成组合 hashCode，常用于重写 hashCode
Objects.isNull(obj);            // 等价于 obj == null
Objects.nonNull(obj);           // 等价于 obj != null
Objects.requireNonNull(obj);    // 为 null 则抛 NullPointerException
Objects.requireNonNull(obj, "obj 不能为空"); // 自定义错误信息
```

`Objects.equals(a, b)` 比 `a.equals(b)` 更安全，避免调用方为 null 时的 NPE；`Objects.requireNonNull(obj, "msg")` 是方法入参校验的标准写法；`Objects.hash(...)` 是重写 `hashCode()` 的简洁写法。

## StringBuilder 和 StringBuffer

String 对象是常量，增删改操作会丢弃原对象并创建新对象，频繁操作会产生大量副本、降低效率；StringBuilder 和 StringBuffer 用可变字符序列解决了这个问题。

| 类 | 特性 | 效率 | 线程安全 | 使用场景 |
| --- | --- | --- | --- | --- |
| String | 不可变字符序列，常量池复用率高 | 低（每次修改都产生新对象） | — | 字符串很少被修改，如配置信息 |
| StringBuffer | 可变字符序列 | 较高 | 是（方法带 `synchronized`） | 大量修改字符串 + 多线程 |
| StringBuilder | 可变字符序列，API 与 StringBuffer 兼容 | 最高 | 否 | 大量修改字符串 + 单线程（优先选用） |

StringBuffer 继承自 `AbstractStringBuilder`（而非直接继承 Object），内部 `byte[] value` 数组无 final 修饰（区别于 String 的不可变），实现了 `Serializable` 接口可串行化，字符串实际存放在堆内存中。

**常用构造器：**

| 构造器                       | 作用                   |
| ---------------------------- | ---------------------- |
| `StringBuffer()`             | 初始容量为 16 个字符   |
| `StringBuffer(int capacity)` | 指定初始容量           |
| `StringBuffer(String str)`   | 初始化为指定字符串内容 |

**常用方法：**

| 方法                                      | 作用                       |
| ----------------------------------------- | -------------------------- |
| `int length()`                            | 获取有效长度               |
| `int capacity()`                          | 返回容器容量               |
| `append(...)`                             | 将内容追加到容器末尾       |
| `delete(int start, int end)`              | 删除 [start, end) 区间内容 |
| `replace(int start, int end, String str)` | 替换指定区间为新字符串     |
| `reverse()`                               | 反转字符串                 |
| `insert(int offset, String str)`          | 在指定索引处插入字符串     |

相互转化：`new StringBuffer(str)` 完成 String → StringBuffer，`sb.toString()` 完成 StringBuffer → String。StringBuilder 用法与之相同。

## Math 类和 System 类

Math 和 System 都属于 `java.lang` 包，用 final 修饰不可被继承，不可被实例化。Math 提供两个公有静态常量 `Math.E`（自然对数）和 `Math.PI`（圆周率）。

**Math 常用方法：**

| 方法 | 描述 |
| --- | --- |
| `abs(x)` | 绝对值，支持 int/long/float/double |
| `pow(a, b)` | a 的 b 次方 |
| `ceil(x)` | 向上取整 |
| `floor(x)` | 向下取整 |
| `round(x)` | 四舍五入，float→int，double→long |
| `sqrt(x)` | 算术平方根 |
| `random()` | 返回 [0.0, 1.0) 的随机数。获取 [a, b] 的随机整数：`(int)(a + Math.random() * (b - a + 1))` |
| `max(a, b)` | 两个数中的最大值，支持 int/long/float/double |
| `min(a, b)` | 两个数中的最小值，支持 int/long/float/double |

多值比较可嵌套调用 `Math.max()`，也可排序后取首尾，或用 Stream API：`Arrays.stream(nums).max().getAsInt()`。

**System 常用方法：**

| 方法 | 描述 |
| --- | --- |
| `exit(int status)` | 退出当前程序，status=0 表示正常退出 |
| `arraycopy(src, srcPos, dest, destPos, length)` | 将 src 数组从 srcPos 索引起的 length 个元素拷贝到 dest 数组的 destPos 位置。目标数组长度不足抛 ArrayIndexOutOfBoundsException，类型不兼容抛 ArrayStoreException |
| `currentTimeMillis()` | 返回当前时间距 1970-01-01 00:00:00 的毫秒数 |

## Date 类和 Calendar 类

| 类 | 引入版本 | 可变性 | 格式化方式 |
| --- | --- | --- | --- |
| Date（java.util，注意与 java.sql.Date 区分） | Java 1.0 | 可变 | `SimpleDateFormat`：`format()` 转字符串，`parse()` 转 Date（需处理 ParseException） |
| Calendar（抽象类） | Java 1.1 | 可变 | 无原生格式化方法，需自行拼接字段 |
| LocalDate / LocalTime / LocalDateTime | Java 8 | 不可变，线程安全 | `DateTimeFormatter.ofPattern(pattern)`（构造器私有） |

- `Date()` 创建当前系统时间对象，`Date(long date)` 按指定毫秒数创建；`getTime()` 将日期对象转换为毫秒值
- `Calendar.getInstance()` 获取实例；`get(field)` / `set(field, value)` 读写日历字段；`Calendar.MONTH` 取值 0-11，需 +1 才是真实月份
- `LocalDate`（年月日）/ `LocalTime`（时分秒）/ `LocalDateTime`（年月日时分秒）解决了 Date/Calendar 的局限：可变性、偏移性（年份从 1900 开始、月份从 0 开始）、Calendar 无格式化方法、线程不安全且不能处理闰秒；`plusDays(long days)` / `minusDays(long days)` 等方法用于日期加减运算

## 八大包装类

基本数据类型不是对象，无法调用类的方法，Java 为八大基本类型提供了对应的包装类，包装类属于 `java.lang` 包。除 Integer 和 Character 外，其余六个包装类名为基本类型首字母大写；除 Boolean 和 Character 外，其余六个都继承自 **Number** 类。

**装箱与拆箱：** JDK 5.0 起支持自动装拆箱，之前需用 `valueOf()` 手动装箱、`intValue()` 等方法手动拆箱。

```Java
Integer a = 100;                  // 自动装箱
Integer b = Integer.valueOf(100); // 手动装箱
int c = a;                        // 自动拆箱
int d = a.intValue();             // 手动拆箱
```

**与 String 类型的转化：**

| 方向 | 方法 |
| --- | --- |
| String → 基本类型 | 包装类的 `parseXxx(String)`（Character 除外）；字符串转 char 用 `charAt(index)` 或 `toCharArray()` |
| 基本类型 → String | 与空字符串拼接，或 `String.valueOf()` |
| String → 包装类 | `parseXxx()`（利用自动装箱）或包装类构造器 |
| 包装类 → String | 与空字符串拼接、`toString()`，或 `String.valueOf()` |

**数值型包装类通用方法（Byte / Short / Integer / Long / Float / Double）：**

| 方法 | 类型 | 作用 |
| --- | --- | --- |
| `xxxValue()` | 实例 | 转换为对应基本类型 |
| `static compare(x, y)` | 静态 | 比较两个基本类型变量大小 |
| `compareTo(AnotherWrapper)` | 实例 | 比较当前对象与另一同类对象大小 |
| `static parseXxx(String)` | 静态 | 字符串解析为基本类型 |
| `toString()` | 实例 | 转为 String |
| `static toString(xxx)` | 静态 | 基本类型值转为 String |
| `static valueOf(...)` | 静态 | 字符串或基本类型转为包装类对象 |
| `static max(x, y)` / `min(x, y)` | 静态 | 最大值 / 最小值 |
| `static sum(x, y)` | 静态 | 两数之和（Boolean/Byte/Short 无此方法） |

**Boolean 类特殊方法：** `booleanValue()` 转为 boolean；`parseBoolean(String)` 只有 "true"（忽略大小写）返回 true，其余返回 false；`compare(x, y)` 相同返回 0，第一个为 true 返回 1，第一个为 false 返回 -1。

**Character 类（最特殊，无 parse/max/min/sum）：** `charValue()` 转为 char；`compare(x, y)` 返回 ASCII 码差值；另有 `isDigit()`、`isLetter()`、`isUpperCase()`、`isLowerCase()`、`isWhitespace()`、`toUpperCase()`、`toLowerCase()` 等字符判断和转换方法。
