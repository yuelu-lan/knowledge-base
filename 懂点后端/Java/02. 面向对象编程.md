Java 是**基于类的静态强类型** OOP 语言，所有类型在编译期检查。

| 特性     | Java                                    |
| -------- | --------------------------------------- |
| OOP 模型 | 基于类（Class-based）                   |
| 类型系统 | 静态强类型，编译期检查                  |
| 类定义   | `class` 是一等公民，语言原生支持        |
| 封装     | `private`、`protected`、`public` 关键字 |
| 继承     | `class Dog extends Animal`，**单继承**  |
| 多态     | 编译期类型检查 + 运行时方法绑定         |
| 抽象类   | `abstract class` 原生支持               |
| 接口     | `interface` + `implements`，编译期强制  |

### 类

#### 构造方法

构造方法是创建对象时自动调用的特殊方法，用于初始化对象的状态。

#### static 成员

被 `static` 修饰的成员属于类本身，而非某个实例，所有对象共享同一份静态成员。

静态代码块在类加载时执行一次，用于初始化静态资源。

| 访问方   | 静态变量 | 成员变量             |
| -------- | -------- | -------------------- |
| 静态方法 | ✅       | ❌（需通过对象引用） |
| 实例方法 | ✅       | ✅                   |

### 封装

封装（Encapsulation）的核心思想是：**把数据隐藏在对象内部，只暴露必要的操作接口**。

Java 提供四个级别的访问修饰符：

| 修饰符                | 本类 | 同包 | 子类（不同包） | 其他类（不同包） |
| --------------------- | ---- | ---- | -------------- | ---------------- |
| `private`             | ✅   | ❌   | ❌             | ❌               |
| `default`（无修饰符） | ✅   | ✅   | ❌             | ❌               |
| `protected`           | ✅   | ✅   | ✅             | ❌               |
| `public`              | ✅   | ✅   | ✅             | ✅               |

- 标准做法：字段用 `private`，对外提供 `public` 的 getter 和 setter

### 继承

- Java 和 JS 都用 `extends` **单继承**，语法基本一致，但 Java 有编译期类型检查更严格
- `super()` 调用父类构造方法，**必须在第一行**
- 方法重写用 `@Override` 注解标记，不写也可以用，但写了编译器会帮你检查签名是否正确
- `final` 可以修饰类（不能被继承）、方法（不能被重写）、变量（赋值后不可修改）
- 所有类隐式继承 `java.lang.Object`

#### 向上转型与向下转型

```java
// 向上转型：子类 → 父类（安全，自动完成）
Animal animal = new Dog();

// 向下转型：父类引用 → 子类（需要显式强转，可能不安全）
Dog dog = (Dog) animal;

// 安全的向下转型：先用 instanceof 检查
if (animal instanceof Dog) {
    Dog dog = (Dog) animal;
    dog.bark();
}
```

### 多态

- 多态 = 父类引用指向子类对象（向上转型）+ 方法重写，同一方法调用在不同子类上有不同行为
- Java 变量有**编译类型**和**运行类型**，实例方法根据运行类型绑定，其他的都为编译类型
- 向下转型必须用 `instanceof` 先检查类型安全，否则运行时抛 `ClassCastException`（类转换异常）

Java 的绑定规则：

- `static` 方法、`private` 方法、`final` 方法、字段访问 → **静态绑定**（编译期确定）
- 普通的实例方法 → **动态绑定**（运行期确定）

> Java 多态中一个非常容易踩的坑！**静态方法没有多态行为**，调用哪个类的静态方法由**变量的声明类型**决定。

多态的实际应用：策略模式 —— 把可变的行为抽象成接口，运行时动态选择不同的实现。

### 抽象

- 抽象类：不能直接 `new`，可以有字段、构造方法、普通方法和抽象方法
- 优先使用接口（更灵活），需要共享字段/状态时用抽象类，两者可以组合

| 语法点     | Java                                          |
| ---------- | --------------------------------------------- |
| 抽象类定义 | `abstract class` 关键字，不能被实例化         |
| 抽象方法   | `abstract` 修饰，无方法体，子类必须实现       |
| 构造方法   | 抽象类可以有构造方法，子类通过 `super()` 调用 |
| 混合实现   | 抽象类可以有抽象方法 + 普通方法               |

抽象类最经典的应用就是模板方法模式 —— 在父类中定义算法骨架，将某些步骤延迟到子类实现。

### 接口

- 接口是纯粹的抽象，只定义"能做什么"，不定义"怎么做"；编译器强制实现所有方法
- 一个类可实现多个接口，用接口实现多继承效果
- 设计意图：接口定义"能力"（has-ability），抽象类定义"本质"（is-a）

| 语法点   | Java                                      |
| -------- | ----------------------------------------- |
| 接口定义 | `interface` 关键字，定义抽象方法契约      |
| 实现接口 | `implements` 关键字，一个类可实现多个接口 |
| 多实现   | ✅ 一个类可实现多个接口                   |
| 默认方法 | `default` 关键字，接口可以有方法实现      |
| 静态方法 | ✅ 接口可以有静态方法                     |
| 常量定义 | 接口中的变量默认是 `public static final`  |

接口的实际应用：依赖倒置原则（DIP） —— 高层模块不应该依赖低层模块，两者都应该依赖抽象。

例如：`UserService` 只依赖 `UserRepository` 接口，不关心底层是 MySQL 还是 Redis —— 切换存储实现时，业务层代码零修改。

#### 接口 vs 抽象类

| 对比项       | 接口                        | 抽象类                      |
| ------------ | --------------------------- | --------------------------- |
| 状态（字段） | 只能有常量，不能有实例字段  | 可以有实例字段              |
| 构造方法     | 没有                        | 有，子类通过 `super()` 调用 |
| 多继承       | 一个类可实现多个接口        | 一个类只能继承一个抽象类    |
| 默认实现     | Java 8+ 支持 `default` 方法 | 一直支持具体方法            |
| 设计意图     | 定义"能做什么"（能力）      | 定义"是什么"（本质）        |

#### 为什么需要 default 方法？

Java 8 之前，接口中的方法**全部是抽象方法**，没有任何实现。这意味着一旦接口发布并被大量类实现，**就无法再向接口中添加新方法**——否则所有实现类都必须修改，编译报错。

### 枚举

- 枚举是一种特殊的类，用于定义一组固定的**常量值**，编译期类型安全
- 枚举本质上是 `Enum` 类的实例，可以有字段、方法和构造方法
- 枚举原生支持 `switch` 和 `values()` 遍历
- 枚举可以实现接口，每个枚举值可以有不同的方法实现

### 重写与重载

- 重写：子类重新定义父类方法，方法签名必须一致，运行时动态绑定，是多态的表现
- 重载：同一个类中定义多个同名但参数不同的方法，编译时静态绑定

### 泛型

- 泛型让类型成为参数，编译期进行类型检查，避免运行时类型转换错误
- Java 泛型有类型擦除：泛型信息在编译后会被擦除，运行时不存在
- 通配符：`? extends T`（协变，只读）、`? super T`（逆变，只写）
- 泛型方法：需要在返回值前声明 `<T>`

#### 通配符

通配符是 Java 泛型中最难理解的部分。记住这个原则（PECS）：

- Producer Extends：如果只需要读取数据（生产者），用 `? extends T`
- Consumer Super：如果只需要写入数据（消费者），用 `? super T`

同时使用两种通配符的经典例子：

```java
// 从 src 读数据，往 dest 写数据
public <T> void copy(List<? extends T> src, List<? super T> dest) {
    for (T item : src) {
        dest.add(item);
    }
}
```

`? extends T`** — 只读（生产者）**

计算任意数字列表的总和：

```java
public double sum(List<? extends Number> list) {
    double total = 0;
    for (Number n : list) {
        total += n.doubleValue();
    }
    return total;
}

sum(List.of(1, 2, 3));        // List<Integer> ✅
sum(List.of(1.5, 2.5, 3.5)); // List<Double>  ✅
```

编译器不知道具体是 `Integer` 还是 `Double`，如果允许写入，你可能往 `List<Integer>` 里塞一个 `Double`，类型就乱了。

`? super T`** — 只写（消费者）**

往列表里批量写入整数：

```java
public void fill(List<? super Integer> list, int count) {
    for (int i = 0; i < count; i++) {
        list.add(i);
    }
}

fill(new ArrayList<Integer>()); // ✅
fill(new ArrayList<Number>());  // ✅
fill(new ArrayList<Object>());  // ✅
```

读出来的类型不确定（可能是 `Number`，也可能是 `Object`），编译器只能给你 `Object`，没有意义。

### 内部类

| 类型 | 语法 | 特点 | 使用场景 |
| --- | --- | --- | --- |
| 成员内部类 | 定义在类内部，与字段和方法平级 | 持有外部类引用，可访问外部类所有成员 | 需要访问外部类状态的辅助类 |
| 静态内部类 | `static` 修饰的内部类 | 不持有外部类引用，不能访问外部类实例成员 | 与外部类逻辑关联但不依赖外部类实例 |
| 局部内部类 | 定义在方法内部 | 只在方法内可见，实际开发中较少使用 | 方法内需要临时使用的复杂逻辑 |
| 匿名内部类 | 没有名字的内部类，一次性使用 | 常用于实现接口或继承类的简写形式 | 事件回调、Runnable、Comparator 等 |

- 创建语法：成员内部类用 `outerObj.new Inner()`，静态内部类用 `new Outer.Inner()`
- 成员内部类可访问外部类所有成员（包括 `private`）；非静态内部类隐式持有外部类引用，可能导致**内存泄漏**，**优先使用静态内部类**
- 成员内部类不能定义 `static` 成员（`public static final` 常量除外），静态内部类可以
- 匿名内部类 Java 8+ 可用 Lambda `() -> {}` 简化，类似 JavaScript 的 `() => {}`
