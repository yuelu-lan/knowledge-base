# 反射、动态代理、注解

## 反射

反射（Reflection）允许程序在运行时动态获取类信息、操作对象、访问属性和调用方法，常用于框架开发、工具类、动态代理、序列化等场景。

**获取类对象**：反射的第一步是获取 Class 类对象，三种方式：

- `Class.forName("全类名")` — 最常用，需处理 ClassNotFoundException
- `类名.class` — 类字面量常量
- `对象.getClass()` — 通过实例获取

```Java
Class<?> c1 = Class.forName("com.example.MyClass");
Class<MyClass> c2 = MyClass.class;
Class<? extends MyClass> c3 = obj.getClass();
// 三种方式获取的是同一个 Class 对象（c1 == c2 == c3）
```

注意 `MyClass.class` 与 `MyClass` 的区别：`MyClass` 是类名（类型声明），用于声明变量、创建对象；`MyClass.class` 是 `Class<MyClass>` 类型的对象，代表类的元信息，用于反射操作。

**构造器**

| 方法 | 说明 |
| --- | --- |
| `getConstructors()` | 返回所有 public 构造器数组 |
| `getDeclaredConstructors()` | 返回所有构造器数组（含 private） |
| `getConstructor(Class<?>... parameterTypes)` | 获取指定 public 构造器 |
| `getDeclaredConstructor(Class<?>... parameterTypes)` | 获取指定构造器（含 private） |

获取构造器后可通过 `newInstance()` 创建对象；private 构造器需先调用 `setAccessible(true)`（暴力反射）取消访问检查。

```Java
Constructor<Person> c = Person.class.getDeclaredConstructor(String.class);
c.setAccessible(true); // 暴力反射，访问 private 构造器
Person p = c.newInstance("张三");
```

**成员变量**

| 方法 | 说明 |
| --- | --- |
| `getFields()` | 返回所有 public 成员变量数组 |
| `getDeclaredFields()` | 返回所有成员变量数组（含 private） |
| `getField(String name)` | 获取指定 public 成员变量 |
| `getDeclaredField(String name)` | 获取指定成员变量（含 private） |

Field 类用于取值和赋值：

| 方法 | 说明 |
| --- | --- |
| `void set(Object obj, Object value)` | 为指定对象的该字段赋值 |
| `Object get(Object obj)` | 获取指定对象的该字段值 |
| `setAccessible(true)` | 取消访问检查（暴力反射，private 成员才需要） |

```Java
Class<Person> clazz = Person.class;
Field nameField = clazz.getDeclaredField("name");
nameField.setAccessible(true);
nameField.set(obj, "新值");
```

**成员方法**

| 方法 | 说明 |
| --- | --- |
| `getMethods()` | 返回所有 public 成员方法数组 |
| `getDeclaredMethods()` | 返回所有成员方法数组（含 private） |
| `getMethod(String name, Class<?>... parameterTypes)` | 获取指定 public 方法 |
| `getDeclaredMethod(String name, Class<?>... parameterTypes)` | 获取指定方法（含 private） |

Method 类通过 `invoke(Object obj, Object... args)` 触发执行：参数一为调用该方法的对象，参数二为方法参数，返回值为方法执行结果。

```Java
Method m = Person.class.getDeclaredMethod("privateMethod");
m.setAccessible(true);
Object result = m.invoke(person);
```

**作用与应用场景**：反射的核心用途是做 Java 框架，例如给定任意对象，在不清楚字段的情况下通过反射获取所有字段名称和值。

```Java
public static void saveObject(Object obj) throws IllegalAccessException {
    Class<?> c = obj.getClass();
    Field[] fields = c.getDeclaredFields();
    for (Field field : fields) {
        field.setAccessible(true);
        String name = field.getName();
        Object value = field.get(obj);
        System.out.println(name + " = " + value);
    }
}
```

## 动态代理

代理模式（Proxy Pattern）的核心思想：不直接访问目标对象，而是通过一个"中间人"来访问，代理可在调用前后插入额外逻辑（日志、权限、事务等）。

```
调用方 → 代理对象 → 真实对象
```

**静态代理**：手动编写代理类，实现与目标对象相同的接口，在方法中包装目标调用；缺点是有多少接口就要写多少代理类，太繁琐。

```Java
class HelloProxy implements Hello {
    private Hello target;
    HelloProxy(Hello target) { this.target = target; }
    public void say(String name) {
        System.out.println("[日志] 调用前");
        target.say(name);
        System.out.println("[日志] 调用后");
    }
}
```

**动态代理**：在运行时自动生成代理类，不需要手写，Java 标准库通过 `java.lang.reflect.Proxy` 内置支持。

3 个关键角色：

| 角色 | 类/接口 | 职责 |
| --- | --- | --- |
| 代理工厂 | `Proxy` | 运行时生成代理类 |
| 拦截处理器 | `InvocationHandler` | 定义拦截到方法调用后的处理逻辑 |
| 方法描述 | `Method` | 代表被调用的方法 |

`Proxy.newProxyInstance()` 参数：

| 参数 | 类型 | 作用 |
| --- | --- | --- |
| `loader` | `ClassLoader` | 定义代理类的类加载器 |
| `interfaces` | `Class<?>[]` | 代理类需要实现的接口列表 |
| `h` | `InvocationHandler` | 方法调用的拦截处理器 |

`InvocationHandler.invoke()` 参数：`proxy`（代理对象本身，很少用）、`method`（当前被调用的方法）、`args`（调用参数数组）。

```Java
UserService proxy = (UserService) Proxy.newProxyInstance(
    target.getClass().getClassLoader(),
    target.getClass().getInterfaces(),
    (proxyObj, method, args) -> {
        System.out.println("[日志] " + method.getName() + " 开始");
        Object result = method.invoke(target, args);
        System.out.println("[日志] " + method.getName() + " 结束");
        return result;
    }
);
```

JDK 动态代理的核心限制：只能代理接口，不能代理普通类，因为生成的代理类 `$Proxy0` 已经继承了 `Proxy`（Java 单继承），只能通过实现接口来代理。

## 注解

注解（Annotation，JDK 5.0 引入）是一种可被编译器打包进 class 文件的"元数据"，用于对类、方法、成员变量等做标记后进行特殊处理，例如 JUnit 的 `@Test` 标记方法会被当作测试方法执行。

**自定义注解**

```Java
public @interface 注解名称 {
    public 属性类型 属性名() default 默认值;
}
```

特殊属性 value：注解只有一个 value 属性时，使用可省略 `value=`；有多个属性且无默认值时，value 名称不能省略。

常见注解形式：

- **标记注解**（无参数）：如 `@Override`、`@Deprecated`
- **单值注解**（1 个参数，通常命名为 value）：如 `@Label("用户ID")`
- **多值注解**（多个参数）：如 `@Table(name="t_user", schema="app", comment="用户表")`
- **带默认值的注解**：部分或全部属性有默认值

**元注解**：用来注解注解的注解，两个核心元注解：

| 元注解 | 作用 | 可选值 |
| --- | --- | --- |
| `@Target` | 约束注解能用在哪些地方 | ElementType：TYPE（类/接口）、FIELD（成员变量）、METHOD（成员方法）、PARAMETER（方法参数）、CONSTRUCTOR（构造器）、LOCAL_VARIABLE（局部变量） |
| `@Retention` | 声明注解的生命周期 | RetentionPolicy：SOURCE（仅源码阶段）、CLASS（源码+字节码，默认值）、RUNTIME（源码+字节码+运行阶段，开发常用） |

**注解的解析**：判断是否存在注解，存在则解析出内容。`Annotation` 是所有注解的顶级接口；`AnnotatedElement` 定义了注解解析方法，Class、Method、Field、Constructor 都实现了该接口。

| 方法 | 说明 |
| --- | --- |
| `getDeclaredAnnotations()` | 获取当前对象上的所有注解 |
| `getDeclaredAnnotation(Class<T>)` | 根据类型获取对应注解对象 |
| `isAnnotationPresent(Class<Annotation>)` | 判断是否使用了指定注解 |

解析技巧：注解在哪个成分上，就先获取哪个成分对象，再获取其上的注解，例如注解在方法上就先获取 Method 对象，再通过 `method.getAnnotation(MyTest.class)` 获取。

**实际应用示例**：自定义 `@MyTest` 注解 + 测试运行器，模拟 JUnit 的核心机制。

```Java
// 自定义注解
@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
public @interface MyTest {
    String value() default "";
}

// 测试运行器：扫描并执行所有带 @MyTest 的方法
public class MyTestRunner {
    public static void run(Class<?> testClass) {
        Method[] methods = testClass.getDeclaredMethods();
        for (Method method : methods) {
            if (method.isAnnotationPresent(MyTest.class)) {
                MyTest myTest = method.getAnnotation(MyTest.class);
                Object instance = testClass.getDeclaredConstructor().newInstance();
                method.setAccessible(true);
                method.invoke(instance);
            }
        }
    }
}
```

这是反射 + 注解组合使用的典型示例，也是 Spring、JUnit 等框架的核心实现原理。
