# 异常处理

## 受检异常 vs 非受检异常

Java 异常体系的独特之处在于**受检异常（Checked Exception）**机制。

**异常层级：**

```plain
Throwable
├── Error（JVM 级别，通常不捕获）
└── Exception
    ├── RuntimeException（非受检，无需声明）
    │   ├── NullPointerException
    │   ├── IllegalArgumentException
    │   └── ArrayIndexOutOfBoundsException
    └── 其他 Exception 子类（受检，必须声明或捕获）
        ├── IOException / FileNotFoundException
        ├── SQLException
        └── ParseException 等
```

| 类型 | 是否需要声明 | 是否需要捕获 | 场景 |
| --- | --- | --- | --- |
| 非受检（RuntimeException 子类） | 不需要 | 可选 | 编程错误 |
| 受检（Exception 其他子类） | **必须** | **必须** | 可预期的外部失败 |

受检异常由编译器强制要求处理，方法签名中必须用 `throws` 声明或在调用处 `try-catch` 捕获。

```java
// 声明 throws：告知调用者可能抛出受检异常
static String readFile(String path) throws IOException {
    throw new IOException("文件未找到: " + path);
}

// 非受检异常：无需声明，可选捕获
try {
    String s = null;
    int len = s.length();
} catch (NullPointerException e) { ... }

// 受检异常：必须 try-catch 或 throws 声明
try {
    readFile("/nonexistent.txt");
} catch (IOException e) { ... }
```

## try-catch-finally 基础

- `finally` 无论是否异常都会执行（用于资源清理）
- `e.getMessage()` — 获取异常消息
- `e.getClass().getSimpleName()` — 获取异常类名
- **异常链（Cause）：** 可以把一个异常包装成另一个，通过 `getCause()` 获取原始原因

## 多异常捕获——Java 7+ 的改进

**多个 catch 块**可针对不同异常类型做不同处理，**子类 catch 必须在父类之前**，否则编译错误。

```java
try { ... }
catch (NumberFormatException e) { /* 格式错误 */ }
catch (ArithmeticException e)   { /* 算术错误 */ }
catch (Exception e)             { /* 兜底 */ }
```

**Java 7+ 多异常合并**——当多种异常处理逻辑相同时，用 `|` 合并更简洁：

```java
catch (NumberFormatException | ArithmeticException e) { ... }
```

## 自定义异常——携带业务信息

自定义异常可携带更丰富的业务上下文信息。

| 类型       | 继承自             | 场景               | 是否强制处理 |
| ---------- | ------------------- | ------------------ | ------------ |
| 受检异常   | `Exception`        | 调用者可合理恢复   | 是           |
| 非受检异常 | `RuntimeException` | 编程错误，无法恢复 | 否           |

**实际建议：** 现代 Java 开发倾向于使用非受检异常（继承 RuntimeException），因为受检异常在 Lambda/Stream 中很难处理。

```java
// 受检自定义异常
class UserNotFoundException extends Exception {
    private final int userId;
    UserNotFoundException(int userId) {
        super("用户不存在，ID: " + userId);
        this.userId = userId;
    }
    int getUserId() { return userId; }
}

// 非受检自定义异常（更常见）
class ValidationException extends RuntimeException {
    private final String field;
    ValidationException(String field, String message) {
        super("字段 [" + field + "] 验证失败: " + message);
        this.field = field;
    }
    String getField() { return field; }
}

static void findUser(int id) throws UserNotFoundException {
    if (id <= 0) throw new UserNotFoundException(id);
}

static void validateAge(int age) {
    if (age < 0 || age > 150)
        throw new ValidationException("age", "年龄必须在 0-150 之间，实际: " + age);
}
```

## try-with-resources——自动资源管理

Java 7 引入的 `try-with-resources` 确保资源（数据库连接、文件流等）使用后自动关闭。

```java
// ✅ try-with-resources（推荐）
try (Connection conn = getConnection()) {
    // 使用 conn
} // 自动调用 conn.close()

// ❌ 传统写法（繁琐，容易遗忘关闭）
Connection conn = null;
try {
    conn = getConnection();
} finally {
    if (conn != null) conn.close();
}
```

资源需实现 `AutoCloseable` 接口（有 `close()` 方法）。多资源时**按声明逆序关闭**（后声明的先关闭）。

```java
// 多资源：按声明逆序关闭（conn2 先关，conn1 后关）
try (Connection conn1 = getConn1();
     Connection conn2 = getConn2()) {
    // 使用 conn1, conn2
}
```

## Lambda 中的异常处理——受检异常的难题

`Function` 的 `apply()` 不声明 `throws`，因此 Lambda 无法直接抛出受检异常，这是 Java 开发的常见痛点。

```java
// ❌ 编译错误：parseAndValidate throws Exception
list.stream().map(s -> parseAndValidate(s));
```

| 方案                  | 适用场景               | 推荐度     |
| --------------------- | ---------------------- | ---------- |
| Lambda 内部 try-catch | 简单场景               | ⭐⭐⭐     |
| 抽取辅助方法          | **最推荐，可读性最好** | ⭐⭐⭐⭐⭐ |
| 通用包装器 wrap()     | 大量复用同一受检函数   | ⭐⭐⭐     |

```java
static String parseAndValidate(String s) throws Exception {
    if (s == null || s.isBlank()) throw new Exception("输入不能为空");
    int n = Integer.parseInt(s);
    if (n < 0) throw new Exception("数值不能为负: " + n);
    return "有效值: " + n;
}

// 方案1：Lambda 内部 try-catch
list.stream().map(s -> {
    try { return parseAndValidate(s); }
    catch (Exception e) { return "❌ " + e.getMessage(); }
})

// 方案2：辅助方法（推荐）
static String safeParse(String s) {
    try { return parseAndValidate(s); }
    catch (Exception e) { return "❌ " + e.getMessage(); }
}
list.stream().map(MyClass::safeParse)

// 方案3：通用包装器
@FunctionalInterface interface CheckedFn<T, R> { R apply(T t) throws Exception; }
static <T, R> Function<T, R> wrap(CheckedFn<T, R> f) {
    return t -> {
        try { return f.apply(t); }
        catch (Exception e) { throw new RuntimeException(e); }
    };
}
```
