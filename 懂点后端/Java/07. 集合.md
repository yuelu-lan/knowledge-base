# 集合

Java 集合框架（Collections Framework）是一套数据容器工具箱，核心是按需求选类：快速查找、频繁增删、还是键值映射，不同集合类性能差异很大。Collection 接口是单列集合的根接口，有 List（有序可重复有索引）、Set（不重复）、Queue（队列）三个子接口；Map 是双列集合，键不能重复。

## Collection

Collection 是单列集合的父类，规定的方法全部单列集合都会继承。

**特点：** List 系列（ArrayList、LinkedList）有序、可重复、有索引；Set 系列中 HashSet 无序不重复无索引，LinkedHashSet 有序不重复无索引，TreeSet 按大小默认升序排序且不重复无索引；Queue 先进先出（FIFO）、有序、可重复、无索引。

**常见方法：**

| 方法 | 说明 |
| --- | --- |
| `add(E e)` | 添加元素 |
| `clear()` | 清空所有元素 |
| `remove(E e)` | 删除指定元素 |
| `contains(Object obj)` | 判断是否包含指定对象 |
| `isEmpty()` | 判断是否为空 |
| `size()` | 返回元素个数 |
| `toArray()` | 转为数组 |
| `addAll(Collection c)` | 将另一个集合的数据全部加入（前提是数据类型一致） |

**遍历方式：** 除了 for 循环，专用方式是迭代器（Iterator），`hasNext()` 判断是否有下一个元素，`next()` 取出当前元素并后移：

```Java
Iterator<String> it = c.iterator();
while (it.hasNext()) {
    String s = it.next();
    System.out.println(s);
}
```

增强 for 本质是迭代器遍历的简化写法；JDK 8 起也可用 `forEach(Consumer)` 配合 Lambda 遍历，如 `c.forEach(System.out::println)`。

## List

List 系列有序、可重复、有索引，ArrayList 和 LinkedList 都具备这些特点，但底层实现不同，适合场景也不同。因支持索引，List 比 Collection 多了以下方法：

| 方法 | 说明 |
| --- | --- |
| `add(int index, E element)` | 在指定位置插入元素 |
| `remove(int index)` | 删除指定索引的元素，返回被删除的元素 |
| `set(int index, E element)` | 修改指定索引的元素，返回被修改的元素 |
| `get(int index)` | 返回指定索引的元素 |

**ArrayList 底层原理：** 基于数组实现。

- 查询快（按索引 O(1)），删除慢（需前移后续元素），添加也慢（可能后移或扩容）
- 无参构造器创建的是长度为 0 的数组，添加第一个元素时创建长度为 10 的数组
- 存满后按 1.5 倍扩容；一次添加多个元素若 1.5 倍放不下，按实际需要扩容
- 适合：按索引随机查询、尾部追加、数据量固定或只增不删、整体遍历、排序和二分查找
- 不适合：中间/头部频繁插入删除（O(n)）、频繁扩容的超大数据量、FIFO 队列操作（改用 ArrayDeque）、频繁判断元素是否存在（改用 HashSet）

**LinkedList 底层原理：** 基于双向链表实现，结点在内存中不连续，各自持有数据值和前后结点引用。

- 查询慢（需从头遍历），增删相对快，首尾元素的增删改查速度极快
- 适合：头部/中间频繁插入删除、实现栈（push/pop O(1)）、实现队列（offer/poll O(1)）、实现双端队列

首尾操作方法：

| 方法 | 说明 |
| --- | --- |
| `addFirst(E e)` / `addLast(E e)` | 头部/尾部插入 |
| `getFirst()` / `getLast()` | 获取首/尾元素 |
| `removeFirst()` / `removeLast()` | 删除并返回首/尾元素 |

**ListIterator：** ArrayList、LinkedList 均可获取，支持双向遍历和操作：

| 方法 | 说明 |
| --- | --- |
| `next()` | 返回右侧元素，游标右移 |
| `previous()` | 返回左侧元素，游标左移 |
| `remove()` | 删除最近一次 next/previous 跨过的元素 |
| `add(x)` | 在游标处插入元素，游标落在新元素右边 |
| `set(x)` | 替换最近一次 next/previous 跨过的元素，游标不变 |

`remove()` 调用规则：只能在 next()/previous() 之后调用，刚创建迭代器、add() 之后、remove() 之后都不能连续调用（否则抛 IllegalStateException）；set() 之后可以接 remove()，但 remove() 之后不能接 set()。

## Queue

Queue 继承自 Collection，支持两组语义相同但失败表现不同的方法：

| 操作 | 抛异常 | 返回值 |
| --- | --- | --- |
| 插入尾部 | `add(e)` | `offer(e)` |
| 获取并删除队首 | `remove()` | `poll()` |
| 获取但不删除队首 | `element()` | `peek()` |

**Deque（双端队列）：** 继承自 Queue，头尾两端都支持增删查，当作 FIFO 队列使用时从尾部添加、从头部删除。

**ArrayDeque：** 底层用循环数组实现，数组任意一点都可能被看作起点或终点，head 指向首端第一个有效元素，tail 指向尾端第一个可插入的空位。核心方法为 addFirst/addLast 在两端插入，pollFirst/pollLast 删除并返回两端元素，peekFirst/peekLast 只查看不删除。非线程安全，不允许放入 null。

**PriorityQueue：** 保证每次取出的都是权值最小的元素，比较依据是自然顺序或构造时传入的 Comparator。底层是完全二叉树实现的小顶堆（任意非叶子节点值不大于左右子节点），用数组存储，不允许放入 null。add/offer 插入并上浮，remove/poll 取出并删除队首，element/peek 只查看队首；每组前者失败抛异常，后者返回 null。

**应用场景：**

- 双端队列：浏览器前进/后退、撤销/重做、滑动窗口算法
- 优先队列：任务调度、Top K 问题、Dijkstra 最短路径、合并 K 个有序链表

## Set

Set 无序、不重复、无索引；自身没有额外新增常用方法，直接用 Collection 的方法。

**HashSet 底层原理：** 每个对象都有一个 int 类型哈希值（`hashCode()` 获取），规则是 `equals()` 为 true 则哈希值必须相同，哈希值相同不代表 `equals()` 为 true。底层是哈希表（JDK 8 前为数组+链表，JDK 8 起为数组+链表+红黑树），数组每个元素称为桶，用哈希码对桶总数取余定位桶，桶为空直接插入，桶不为空（哈希碰撞）则与桶内对象逐个比较后插入链表。装填因子默认 0.75，元素数超过桶总数的 75% 时双倍扩容并 rehash，桶数默认 16、取 2 的幂；当某个桶的链表长度超过 8 且数组长度 >= 64 时，链表转成红黑树以优化查找。默认不能对内容相同的不同对象去重，需按内容去重必须重写 `hashCode()` 和 `equals()`。

**LinkedHashSet：** 底层同样是哈希表，但每个元素额外维护一个双向链表记录插入顺序，本质是"不允许重复 + 保持插入顺序"的 Set，适合保序去重场景。

**TreeSet：** 不重复、无索引、默认升序排序，底层基于红黑树实现。数值按大小、字符串按首字符编号排序，自定义类型必须指定排序规则——让类实现 Comparable 重写 compareTo，或创建时传入 Comparator。判断重复不用 equals()，只看比较结果是否为 0，因此排序字段组合要能唯一标识元素。

TreeSet 额外方法：

| 方法 | 说明 |
| --- | --- |
| `first()` / `last()` | 返回最小/最大元素 |
| `higher(v)` / `lower(v)` | 返回严格大于/小于 v 的最近元素 |
| `ceiling(v)` / `floor(v)` | 返回大于等于/小于等于 v 的最近元素 |
| `pollFirst()` / `pollLast()` | 删除并返回最小/最大元素 |
| `descendingSet()` / `descendingIterator()` | 返回递减顺序的集合/迭代器 |

## 并发修改异常（ConcurrentModificationException）

使用迭代器（含增强 for）遍历集合的同时通过集合自身方法增删元素，或多个迭代器同时读写，会触发该异常。原理：集合内部维护修改次数计数器 modCount，迭代器持有期望值 expectedModCount，每次操作前两者不一致就抛异常。

安全删除方式：

- 用迭代器遍历并调用 `it.remove()` 删除
- for 循环倒序遍历删除
- 正序遍历删除后执行 `i--`

## Map

Map 是双列集合，一次存一对键值对（Entry），键不允许重复但值可以重复。特点由键决定：HashMap 无序不重复无索引（最常用），LinkedHashMap 有序不重复无索引，TreeMap 按键大小默认升序排序且不重复无索引。

**常用方法：**

| 方法 | 说明 |
| --- | --- |
| `put(K key, V value)` | 添加/覆盖键值对 |
| `size()` | 获取集合大小 |
| `clear()` | 清空集合 |
| `isEmpty()` | 判断是否为空 |
| `get(Object key)` | 根据键获取值 |
| `remove(Object key)` | 根据键删除键值对（返回被删的值） |
| `containsKey(Object key)` | 判断是否包含某个键 |
| `containsValue(Object value)` | 判断是否包含某个值 |
| `keySet()` | 获取全部键的 Set 集合 |
| `values()` | 获取全部值的 Collection |
| `putAll(Map m)` | 将另一个 Map 的数据全部加入 |

除了 `keySet()`/`entrySet()` 配合迭代器遍历，JDK 8 起也可直接用 `forEach(BiConsumer)`：

```Java
map.forEach((k, v) -> System.out.println(k + " -> " + v));
```

**HashMap：** 底层原理与 HashSet 一模一样（Set 系列本质就是基于 Map 实现，只用键不用值），键依赖 `hashCode()`/`equals()` 保证唯一性，自定义类型作键需重写这两个方法。put 流程：哈希寻址 → 处理哈希冲突（链表或红黑树）→ 判断是否需要扩容 → 插入/覆盖节点。

**LinkedHashMap：** 底层是哈希表，但每个键值对额外维护一个双向链表记录顺序，LinkedHashSet 的底层实现就是 LinkedHashMap。

**TreeMap：** 底层基于红黑树，原理与 TreeSet 一致，支持两种排序方式：键的类实现 Comparable，或构造时传入 Comparator。

## 补充

可变参数格式为 `数据类型...参数名`，可传 0 个、多个或一个数组，方法内部当数组使用；一个形参列表中只能有一个可变参数，且必须放在最后。

**Collections 工具类：**

| 方法 | 说明 |
| --- | --- |
| `addAll(Collection c, T... elements)` | 批量添加元素 |
| `shuffle(List list)` | 打乱 List 元素顺序 |
| `sort(List list)` | 对 List 升序排序（元素须实现 Comparable） |
| `sort(List list, Comparator c)` | 按比较器规则对 List 排序 |

Collections 只支持对 List 排序，用无参 sort 时自定义类型必须实现 Comparable 并指定比较规则。
