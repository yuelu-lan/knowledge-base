### 创建 Stream 的方式

1. **惰性求值** — 中间操作（filter/map）不立即执行，终端操作（collect/forEach）才触发
2. **一次性** — Stream 被终端操作消费后不能再次使用，否则抛 `IllegalStateException`
3. **不修改源** — Stream 不改变原集合，所有操作返回新结果

| 方式          | 代码                        |
| ------------- | --------------------------- |
| 从 Collection | `list.stream()`             |
| 从数组        | `Arrays.stream(arr)`        |
| Stream.of()   | `Stream.of(1,2,3)`          |
| 无限流        | `Stream.generate(supplier)` |
| 迭代生成      | `Stream.iterate(seed, fn)`  |

### 最常用的中间操作

| 操作                  | 描述                                    |
| --------------------- | --------------------------------------- |
| `.filter(predicate)`  | 保留满足条件的元素                      |
| `.map(function)`      | 将元素转换为另一种形式                  |
| `.flatMap(function)`  | 将每个元素映射为一个流，然后展平合并    |
| `.mapToInt(function)` | 转换为 IntStream（支持 sum/average 等） |

### 进阶中间操作

| 操作                 | 描述               |
| -------------------- | ------------------ |
| `distinct()`         | 去重               |
| `sorted()`           | 自然排序           |
| `sorted(Comparator)` | 自定义排序         |
| `limit(n)`           | 取前 n 个          |
| `skip(n)`            | 跳过前 n 个        |
| `peek(fn)`           | 调试用，不修改元素 |

**分页组合：** `skip(pageSize * pageNum).limit(pageSize)` 可实现分页。

### 终端操作——触发实际计算

终端操作是 Stream 流水线的「启动按钮」，调用后 Stream 才真正开始执行。

| 终端操作                  | 描述                     |
| ------------------------- | ------------------------ |
| `forEach(fn)`             | 遍历每个元素             |
| `count()`                 | 返回元素个数             |
| `findFirst()`             | 返回第一个满足条件的元素 |
| `anyMatch(fn)`            | 是否有任意元素满足条件   |
| `allMatch(fn)`            | 是否全部元素满足条件     |
| `noneMatch(fn)`           | 是否没有元素满足条件     |
| `min(comp)` / `max(comp)` | 最小值 / 最大值          |
| `collect()`               | 汇集为集合               |
| `reduce()`                | 规约操作                 |

**findFirst、min、max 返回 **`Optional`** 容器**，通过 `.orElse()` 取值可避免 NPE。

### collect 与 Collectors——强大的汇集操作

`collect()` 是最灵活的终端操作，配合 `Collectors` 工具类可完成各种汇集：

| Collector | 结果 | 描述 |
| --- | --- | --- |
| `toList()` | `List` | 收集为列表 |
| `toSet()` | `Set` | 收集为集合（自动去重） |
| `joining(sep)` | `String` | 字符串拼接 |
| `groupingBy(fn)` | `Map<K, List<V>>` | 按条件分组（类似 SQL GROUP BY） |
| `counting()` | `Long` | 分组后计数 |
| `summarizingInt()` | 统计摘要 | count/sum/min/max/avg |
| `partitioningBy(pred)` | `Map<Boolean, List>` | 按布尔条件二分 |

> **Java 16+：** `stream.toList()` 可替代 `collect(Collectors.toList())`，更简洁，返回**不可变列表**。如需可变列表仍用 `Collectors.toList()`。

### reduce——规约操作

`reduce` 是 Stream 中最底层的归约操作，将流中所有元素按指定逻辑合并为一个值。

| 重载 | 返回类型 | 说明 |
| --- | --- | --- |
| `reduce(identity, accumulator)` | `T` | 有初始值，空流返回初始值，推荐 |
| `reduce(accumulator)` | `Optional<T>` | 无初始值，空流返回 empty |

**重要约束：** accumulator 必须是**无副作用且满足结合律**的，以保证并行流（`parallelStream()`）结果正确。例如加法满足结合律，但字符串拼接在并行流中可能产生不确定顺序。

**实际建议：** 大多数 reduce 场景有更语义化的替代——求和用 `mapToInt().sum()`，最大值用 `max(Comparator)`，字符串拼接用 `Collectors.joining()`。

### 并行流——Stream 的并发加速

Stream 内置**并行处理**支持，只需将 `.stream()` 改为 `.parallelStream()`，或调用 `.parallel()`，即可利用多核 CPU 并行处理数据。

```java
// 串行流
list.stream().filter(...).map(...).collect(...)

// 并行流（自动多线程）
list.parallelStream().filter(...).map(...).collect(...)
```

- accumulator 必须满足**结合律**且**无副作用**
- 数据量小时并行流反而更慢（线程调度开销 > 计算收益），建议数据量 > 10000 时再考虑
- 避免在并行流中修改共享状态（线程不安全），应使用 `collect` 汇集结果
